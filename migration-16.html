16. 表示崩れのチェック：AIを使った全ページ自動検閲

インポート作業が完了した直後、サイト内には思わぬ「ゴミ」や「表示崩れ」が残っているものです。GitHub時代の特殊なタグがそのまま表示されていたり、画像サイズがWordPressのテーマに合わずにはみ出していたり。100記事すべてを自分の目で確認する代わりに、AIとツールを駆使した自動検閲を行いましょう。

1. AI（Vision API）によるレイアウト崩れの検知

プロの現場では、全ページのスクリーンショットを自動で撮影し、AI（Gemini 1.5 Proなど）に解析させる手法が取られます。「この画像群の中で、メニューが重なっていたり、テキストが枠からはみ出しているページをリストアップして」と指示するだけで、修正が必要なページを数秒で特定できます。

2. リンクチェッカーによる「404エラー」の最終狩り

第3章で設定したリダイレクトが正しく動作しているか、リンク切れがないかを最終確認します。「Broken Link Checker」などのプラグインを使えば、サイト内の全てのリンクをクロールし、エラーになっている箇所をレポートしてくれます。

3. GitHub特有の「独自コード」の残骸探し

GitHub Pages（JekyllやHugoなど）で使っていた `{{ site.url }}` などの独自変数が、プレーンテキストとして記事に残っていないか確認してください。これらはデータベース検索で「{{」などの記号をキーワードに検索することで、一括で見つけ出し、WordPress形式へ修正できます。

4. 公開前の「モバイル実機テスト」

ツールでのチェックが終わったら、主要なページ（トップ、人気記事3つ、カテゴリー一覧）だけは、自分のスマートフォンで実際に操作して確認してください。指でタップしやすいか、読み込みスピードにストレスがないか。最後は「ユーザーの感覚」が正解を決めます。

💡 第4章の総仕上げアクション：

サイト全体のリンクチェックを走らせ、エラーが「0」になるまで修正を繰り返しましょう。これで「お引っ越し」は完了です！
