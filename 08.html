8. 独自ドメインの取得とGitHub Pagesへの紐付け

GitHub Pagesが提供するデフォルトのURL（ユーザー名.github.io）でもサイト運営は可能ですが、長期的な「資産」としてサイトを育てるなら、独自ドメインの取得は必須です。独自ドメインを持つことで、万が一プラットフォームを移転することになっても、それまでに積み上げたSEO評価（リンク資産）を失わずに済みます。また、読者からの信頼性も飛躍的に向上します。

ドメイン取得の考え方と選び方

ドメインは「お名前.com」や「バリュードメイン」、「Cloudflare」などのレジストラ（登録業者）で購入できます。選ぶ際のポイントは、サイトのテーマを連想させ、かつ覚えやすい短い名前にすることです。末尾（TLD）は、日本向けであれば「.jp」、汎用性なら「.com」や「.net」が安定していますが、近年は安価な「.xyz」なども普及しています。ただし、更新費用が極端に高いものもあるため、初年度だけでなく継続コストもAIに調査させてから決定しましょう。

GitHub Pagesへの紐付け手順

ドメインを入手したら、GitHub Pagesの設定画面（Settings > Pages）にある「Custom domain」欄に自分のドメインを入力します。しかし、これだけでは繋がりません。ドメインを購入した側の管理画面で「DNS設定」を行う必要があります。主に以下の2つの設定が一般的です。

  • CNAMEレコード: www.あなたのドメイン を ユーザー名.github.io に向ける設定。
  • Aレコード: 独自ドメインそのもの（ネイキッドドメイン）を使用する場合、GitHubが指定する4つのIPアドレスを設定。

設定が反映（浸透）するまでには数時間から最大24時間程度かかることがありますが、正しく紐付くと、あなた専用の「Web上の住所」で世界最速レベルの静的サイトが公開されます。次はこの住所の安全性を保証する「SSL化」について確認していきましょう。
